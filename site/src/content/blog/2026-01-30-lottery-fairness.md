When people ask *‚Äúis the lottery rigged?‚Äù* they usually mean one of two things:

1. **Some numbers appear too often** (frequency intuition).
2. **The process has a systematic bias** (a real mechanism).

This post is a **reproducible** template that runs on **real draw history**.

Here I use **Lebanese Loto (6/42)**, using publicly available draw pages.

All plots in this post are generated by:

- `site/analysis/generate_all.py` (task: `lottery_fairness`)

And the dataset is fetched by:

- `site/analysis/fetch_lebanon_loto.py`

## Step 1 ‚Äî frequency deviations (real draws)

If the machine is fair, each ball should appear about equally often across many draws.

The plot below shows, for each ball 1..42:

- **observed count ‚àí expected count**

![](/blog/2026-01-30-lottery-fairness/freq-deviation-lebanon.svg)

Interpretation:

- Positive bars = ‚Äúhotter than expected‚Äù
- Negative bars = ‚Äúcolder than expected‚Äù
- With finite data, you should expect visible spread.

## Step 2 ‚Äî a simple global test (chi-square) on real data

A classic global test for categorical frequencies is the **chi-square goodness-of-fit test**.

- Null hypothesis: each ball is equally likely (in expectation)
- Test statistic: how far observed counts deviate from expected counts

![](/blog/2026-01-30-lottery-fairness/chi2-summary.svg)

Important caveat:

- A low p-value can indicate bias **or** data issues (scraping errors, missing draws, rule changes, etc.). It‚Äôs a flag, not a verdict.
